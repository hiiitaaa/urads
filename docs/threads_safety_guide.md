# Threads API 安全運用ガイド

API経由の自動投稿でアカウントBANされないための対策集。

## 核心原則（2026-03確認済み実績情報）

**「どれだけ攻めるか」ではなく「どれだけ自然に見せるか」**

### 凍結リスクの具体的な数値目安
- **10分以内に2投稿以上** → 凍結リスク大
- **1時間で10投稿以上** → 凍結リスク大
- **1時間で50〜100以上のアクション（いいね/フォロー）** → 不自然判定
- **新規アカウントは特に厳しい**

### 同じ動きの繰り返し = 自動化判定
- 毎回同じ時間間隔で投稿 → NG
- 同じ文章構成の繰り返し → NG
- 同じリンクを貼り続ける → NG

### リンク投稿の制限
- 毎投稿リンク付き → スパム判定
- 投稿直後に即リンク → スパム判定
- **リンク投稿は全体の2〜3割以内**
- 最低でも5投稿に1回に抑える

### Instagram連鎖
- インスタ側のアクション制限/違反 → Threadsも巻き添え制限
- 完全連動。片方だけ安全にしても意味がない

### 不審ログイン
- 短期間で端末/IPが頻繁に変わる → ロック対象
- 端末と環境はなるべく固定

## 1. 公式レートリミット

| 項目 | 制限 |
|------|------|
| 投稿 | **250件/24時間**（ローリングウィンドウ） |
| リプライ | 投稿とは別カウント |
| カルーセル | 1投稿扱い |
| リンク | 1投稿に最大5個 |
| ハッシュタグ | **1投稿に1個まで** |
| 長期トークン | 60日有効（1日後〜期限前にリフレッシュ） |

**投稿前に必ずクォータ確認:**
```
GET /{user_id}/threads_publishing_limit?fields=quota_usage,config&access_token={TOKEN}
```

## 2. BANされる行動パターン

### 確実にアウト
- **新規アカウントでいきなり大量API投稿**（最も多いBAN原因）
- **短時間のバースト投稿**（数分間に何件も連続投稿）
- **ほぼ同じ文章の繰り返し投稿**（リンクだけ変えるのもNG）
- **毎投稿にリンク**（スパムの典型パターン）
- **エンゲージメントベイト**（「タグ3人」「いいねしてね」等）

### 要注意
- **30分以内にフォロー10-20件 + いいね40-50件** → 実際にBAN報告あり
- **他プラットフォームからのコピペ投稿**（コンテンツフィンガープリント検知される）
- **AI生成文章をそのまま連投**（人間の編集なし）
- **BANされたハッシュタグの使用**
- **Instagram/Facebookアカウントの違反** → Threadsにも連鎖BAN

### 意外な落とし穴
- **テストモードでも投稿は本物**。テスト環境=サンドボックスではない
- **Meta認証（有料）でもBAN保護にならない**。2025年のBAN波ではMeta Verified加入者も大量にBAN
- **複数アカウントを同じIP/デバイスで運用** → 一つのBAN=全アカウント巻き添え

## 3. 安全な投稿頻度

| フェーズ | 頻度 | 備考 |
|---------|------|------|
| アカウントウォーミング（1-2週） | 1回/日、手動 | API使わない。いいね・リプライも手動で5-10/日 |
| ウォーミング（3-4週） | 1-2回/日、API | 手動のエンゲージメントも継続 |
| 通常運用（5週〜） | **3-5回/日** | 4時間以上の間隔を空ける |
| ハードリミット | 250回/24時間 | **絶対に近づかない。20-50回/日を上限に** |

### 投稿間隔の推奨
- 最低 **4時間**（アルゴリズムの60-90分表示ウィンドウを活かす）
- 同時刻に複数予約がある場合、**1-5分のランダム遅延**を追加

---

## 4. コンテンツバリエーション

同じパターンの投稿を繰り返すとスパム判定される。

### やるべきこと
- **フォーマットを回す**: テキスト → 画像 → カルーセル → テキスト
- **文章の長さを変える**: 1-2行の短文と3-5行の長文を混ぜる
- **構成パターンを変える**: 「フック→本文→CTA」を毎回使わない
- **ハッシュタグは付けたり付けなかったり**: 全投稿に付けない
- **人間っぽさを残す**: 自分のコメントに返信、たまに編集（訂正）

### やってはいけないこと
- テンプレートの変数だけ差し替えて量産
- 全投稿にリンクを含める
- 毎投稿同じCTA

---

## 5. シャドウバン検知と回復

### 検知方法
- **設定 → アカウント → アカウントステータス** で確認
- 「おすすめに表示される資格があります」以外 → シャドウバンの可能性
- エンゲージメントが50%以上急落

### 回復方法
1. **48-72時間、全自動化を停止**
2. 手動でのみ投稿（1日1回程度）
3. 回復したら、以前より少ない頻度で再開

---

## 6. トークン管理

- 長期トークンは **30日経過後〜60日前** にリフレッシュ（1回だけ）
- 頻繁なトークン再交換はしない（不審な挙動に見える）
- 1アカウント1トークンを厳守（同時に複数トークン生成しない）
- トークン期限切れ前にアプリ内通知を出す

---

## 7. Uradsでの実装方針

### 投稿時
1. 投稿前に `threads_publishing_limit` でクォータ確認
2. 24時間以内の投稿数が上限（デフォルト20件）を超えていたら拒否
3. 同じアカウントの前回投稿から最低4時間空ける
4. 同時刻の予約は1-5分のランダム遅延を追加

### Cron処理
5. 複数の予約投稿を処理する時、各投稿の間に30-60秒の遅延
6. 429エラー → 指数バックオフ（1分→2分→4分）
7. 401エラー → トークン期限切れ通知
8. コンテンツポリシーエラー → 投稿を失敗にして通知

### トークン
9. 30日後に自動リフレッシュ（Cron）
10. 期限7日前にアプリ内警告表示

### Instagram連鎖BAN対策
11. 設定画面にInstagram連鎖BANの注意書きを表示
