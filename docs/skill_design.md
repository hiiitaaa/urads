# Urads AIアシスタント スキル設計書

## 概要

AIアシスタントは**スキル駆動**で動作する。
ユーザーはチャットパネルのタブバーからスキルを選択。
裏側では明示的にスキルmd を読み込み、専門家ペルソナ + 手順 + ゴール に基づいて実行。
自由チャットとスキルボタンは共存する。

## アーキテクチャ

```
[チャットパネル]
  ┌─────────┬──────────┬──────────┬──────────┐
  │ 投稿分析 │ 競合分析  │ 投稿提案  │ 日次レポ  │  ← タブバー（スキルボタン）
  └─────────┴──────────┴──────────┴──────────┘
  [自由チャット入力欄]
       ↓ ボタンクリック or 自由入力
  スキルmd を読み込み
  Agent SDK の systemPrompt にペルソナ + 手順を注入
  手順通りに Workers API を呼びながら実行
  60点→自己評価→100点の改善ループ
```

## スキル一覧

### 1. 投稿分析（post-analyze）

**ペルソナ**: SNSデータアナリスト + 読者視点の評価者

**目的**: 自分の投稿のパフォーマンスを分析し、何がウケたか・なぜウケたかを要素分解する

**手順**:
1. GET /accounts でアカウント取得
2. GET /posts で直近の投稿一覧取得
3. 各投稿のエンゲージメント（いいね・リプライ・リポスト）を取得
4. パフォーマンスをスコアリング（相対評価）
5. トップ投稿の共通要素を抽出（文体・長さ・時間帯・テーマ・CTA有無）
6. ワースト投稿の問題点を指摘
7. **読者視点**: 読者はなぜこの投稿に反応したのか/しなかったのかを分析
8. 改善可能なポイントを具体的に提示

**出力形式**:
```
## 投稿分析レポート

### スコアランキング
| 順位 | 投稿内容（冒頭20字） | ❤ | 💬 | 🔁 | スコア |
|------|---------------------|---|---|---|--------|

### トップ投稿の成功要素
- 要素1: ...
- 要素2: ...

### 改善ポイント
- ...

### 読者心理分析
- トップ投稿に対して読者は「...」と感じた
- ワースト投稿は「...」のため反応が薄かった
```

### 2. 競合分析（research-competitor）

**ペルソナ**: SNSデータアナリスト（競合調査専門）

**目的**: 競合のバズ投稿を構造化し、成功パターンを抽出する

**手順**:
1. GET /research/benchmarks でベンチマーク一覧取得
2. GET /research/benchmarks/:id/posts?buzz=1 でバズ投稿取得
3. GET /research/benchmarks/:id/analysis でパターン分析取得
4. バズ投稿の共通要素を構造化:
   - フォーマット（テキストのみ/画像付き/カルーセル）
   - 文体（問いかけ型/宣言型/ストーリー型）
   - 長さ（短文/中文/長文）
   - 投稿時間帯
   - テーマ・トピック
   - CTA（Call to Action）の有無と種類
   - ハッシュタグ使用パターン
5. 競合間の比較（複数ベンチマークがある場合）
6. 自分のアカウントとの差分を指摘

**出力形式**:
```
## 競合分析レポート

### バズ投稿パターン
| パターン | 出現率 | 平均エンゲージメント |
|---------|--------|-------------------|

### 成功要素ランキング
1. ...
2. ...

### 自分のアカウントとの差分
- 競合はXXをしているが、自分はYYしていない
- ...
```

### 3. 投稿提案（post-suggest）

**ペルソナ**: 占い師SNSの専門家 + コンテンツストラテジスト

**目的**: 投稿分析 + 競合分析の結果を元に、バズる投稿を3案提案する

**前提**: 投稿分析と競合分析の結果が必要。未実行なら先にそれらを実行する。

**手順**:
1. 投稿分析結果を確認（なければ投稿分析スキルを先に実行）
2. 競合分析結果を確認（なければ競合分析スキルを先に実行）
3. 今バズっている投稿をキーワード検索で取得（GET /research/search）
4. **初回提案（60点の叩き台）**: 3案作成
   - 競合のバズ要素を取り入れつつ、自分のアカウントのトーンに合わせる
   - 「ズラシ」: 競合の丸パクリではなく、自分のオリジナリティを加える
5. **自己評価**:
   - なぜこの投稿がバズると考えるか？根拠は？
   - 読者がこれを見た時の感情の動きは？
   - 自分のアカウントの世界観と一致しているか？
   - Threadsというプラットフォームの特性（500文字、会話型、フェディバース）に最適か？
6. **改善**: 評価を元に3案を修正
7. **最終提案（100点版）**: 改善後の3案 + 各案のバズ予測理由

**出力形式**:
```
## 投稿提案

### 分析サマリ
- 自分の強み: ...
- 競合のバズ要素: ...
- 今のトレンド: ...

### 提案（100点版）

#### 案1
投稿文: 「...」
バズ予測理由: ...
読者の感情の動き: ...
推奨投稿時間: XX:00

#### 案2
（同上）

#### 案3
（同上）

### 採用判断のポイント
- 案1は○○な読者に刺さる
- 案2は○○の場合に有効
- 案3は○○を狙う場合に最適
```

### 4. 日次レポート（daily-report）

**ペルソナ**: SNSマーケティングマネージャー

**目的**: 昨日の成績と今日のアクションプランを提示する

**手順**:
1. GET /posts で昨日の投稿を取得
2. 各投稿のエンゲージメントを確認
3. GET /replies/logs で昨日のリプライ活動を確認
4. GET /research/limits でAPI残量確認
5. 昨日 vs 前日 の比較（可能なら）
6. 今日やるべきことを3つ提案

**出力形式**:
```
## 日次レポート（YYYY-MM-DD）

### 昨日の成績
- 投稿数: X件
- 合計いいね: X
- 合計リプライ: X
- ベスト投稿: 「...」（❤X）

### リプライ自動化
- 自動返信: X件
- ルール稼働状況: ...

### API残量
- 投稿: X/200
- リサーチ: profile X/160, threads X/160

### 今日のアクションプラン
1. ...
2. ...
3. ...
```

## 60点→100点ルール（全スキル共通）

1. 初回出力は「60点の叩き台」として扱う
2. 自己評価: 問題点を列挙する
3. 改善: 問題点を修正して100点に近づける
4. 何が問題でどう直したかを明記する

投稿提案スキルでは特に厳密に適用する。

## スキル間の連携

```
投稿分析 ──→ 投稿提案（分析結果を参照）
競合分析 ──→ 投稿提案（バズ要素を参照）
投稿提案 ──→ 投稿作成（チャットで「1番で投稿して」）
日次レポ ──→ 投稿分析（成績が悪い場合に深堀り）
```

投稿提案は投稿分析と競合分析の結果を前提とする。
未実行の場合は「まず分析を実行しますか？」と確認する。

## Threads プラットフォーム理解（全スキル共通知識）

- 文字数上限: 500文字
- ハッシュタグ: 1投稿1個まで
- 会話型プラットフォーム: 問いかけ・議論がバズりやすい
- フェディバース対応: 他プラットフォームからも見える
- 画像/動画: エンゲージメント向上に有効
- アルゴリズム: ネイティブコンテンツ優遇（他SNSコピペはNG）
- 投稿タイミング: 20-22時がピーク（日本市場）
- BAN回避: 同じ内容の連投禁止、リンクスパム禁止

## 今後追加予定のスキル

- **画像提案**: バズる画像の構図・テキスト配置を提案
- **リプライ戦略**: どの投稿にどんなリプライルールを設定すべきか提案
- **フォロワー分析**: フォロワー属性に基づくコンテンツ戦略
- **トレンド検知**: 今バズっているトピックを検知して即座に投稿提案
