# Buzz Rewrite Skill

バズった投稿を**要素レベル**で分解し、ユーザーの世界観に合わせて**構造を流用・テーマを翻訳**したリライトを生成する。

## 核心原則

1. **丸パクリしない**: 構造は流用、表現は世界観で再構成
2. **テーマ差し替え優先**: 扱わない領域でも、抽象テーマを維持したまま専門領域に翻訳（スキップは最終手段）
3. **NG違反は即スキップ**: ハードルール違反の投稿は proceed=false で中止
4. **Threads特性を守る**: 500字以内、1ハッシュタグ、会話型

## 前提取得

実行前に以下を取得・確認:

- 元投稿（テキスト、likes/replies/reposts、メディア情報、posted_at）
- 世界観テキスト（`ai_settings.rewrite_persona` の内容）
- キャッシュされた要素分解結果（`analysis_results` の `type='post_elements'`、あれば）

世界観が未設定（空）の場合は中止して「先に /persona-brainstorm で世界観を作ってください」と案内する。

## Phase 1: 要素分解

元投稿を以下のJSONに分解:

```json
{
  "format": "リスト型|ストーリー型|問いかけ型|宣言型|TIPS型|比較型|予測型|告白型",
  "hook": "1行目の引き込み方（例: 意外な数字の提示）",
  "core_message": "主張を1文で",
  "structure": "構造パターン（例: フック→根拠×3→CTA）",
  "theme": {
    "specific": "具体テーマ（例: 仕事運）",
    "abstract": "抽象テーマ（例: 運気アップ）"
  },
  "cta": "なし|リプ|保存|DM|フォロー",
  "emotion_trigger": "共感|驚き|好奇心|希望|不安|怒り",
  "tone": "断定|相談|柔らかい|厳しい",
  "length": 287
}
```

**キャッシュがあれば再利用**（analysis_results から scraped_post_id で引く）。

## Phase 2: 適用判定

1. `theme.abstract` を世界観「扱わない領域」と照合
   - マッチ → 専門領域から類似テーマを選択 → `target_theme` に置換
   - 例: `運気アップ@仕事運` → `運気アップ@恋愛運`
2. `core_message` が NG条項に該当 → `proceed: false` で中止
3. NGワードは言い換え候補リストを生成

## Phase 3: 60点ドラフト（2〜3案）

切り口を変えて複数案:

- **案A**: 構造そのまま・テーマだけ差し替え
- **案B**: 構造を世界観の「投稿フォーマット」に合わせて再構成
- **案C**: 決めフレーズで包んだストーリー調

## Phase 4: 自己評価

各案を以下のチェックリストで評価:

- [ ] 世界観のキャラ・口調と一致しているか
- [ ] 決めフレーズ・絵文字ルールを守れているか
- [ ] 元投稿の**丸パクリ度**（5段階、3以上なら書き直し）
- [ ] NGに抵触していないか
- [ ] Threads特性（500字、会話型、1ハッシュタグ）
- [ ] 冒頭1行目に引き込みがあるか
- [ ] CTAは世界観と合っているか

## Phase 5: 100点版

評価結果で修正した最終案2〜3案 + 各案の採用判断ポイント。
AI推しの1案に `is_default: true` を付ける。

## 出力フォーマット

```json
{
  "proceed": true,
  "skip_reason": null,
  "theme_remapped": true,
  "original_theme": "仕事運アップ",
  "target_theme": "恋愛運アップ",
  "variants": [
    {
      "content": "リライト本文",
      "reasoning": "この案の採用判断ポイント（読者にどう刺さるか）",
      "is_default": true,
      "self_eval": {
        "plagiarism_score": 2,
        "tone_match": "high",
        "threads_fit": "ok"
      }
    }
  ],
  "element_analysis": { /* Phase 1のJSONをそのまま */ }
}
```

proceed=false の場合:

```json
{
  "proceed": false,
  "skip_reason": "NG違反: 健康診断の断言を含む",
  "variants": []
}
```

## エッジケース

| ケース | 扱い |
|---|---|
| 画像/動画付き | テキストのみリライト、元メディア情報をメモ付与 |
| 短すぎる投稿（<50字） | スキップ（要素分解精度が出ない） |
| スレッド（連投） | 1投稿目のみ対象 |
| 生成失敗（APIエラー） | 呼び出し側で3回まで再試行、それでも失敗ならスキップ |

## 禁止事項

- 元投稿のフレーズを5語以上連続で流用しない
- 決めフレーズを世界観設定外のものに変えない
- NG該当時に「気をつけて書けばOK」と妥協しない（必ずスキップ）
- 一度に4案以上生成しない（判断疲労回避）
