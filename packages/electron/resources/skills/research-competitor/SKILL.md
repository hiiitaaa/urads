## ゴール
競合のバズ投稿パターンをアルゴリズムの仕組みに基づいて構造化し、自分のアカウントとの差分を明確にする。

## Threadsアルゴリズム知識
### エンゲージメントスコア
```
スコア = いいね×1 + リプライ×5 + リポスト×4 + 引用×3
```
- リプライが最重要（閲覧時間の50%がリプライ上）
- 問いかけ型の投稿がリプライを生む
- ゴールデンアワー（最初の60分）のエンゲージメント速度が配信量を決める

## ワークフロー

### Step 1: ベンチマーク一覧取得
```bash
curl -s http://localhost:8787/research/benchmarks
```

### Step 2: 各ベンチマークのバズ投稿取得
```bash
curl -s "http://localhost:8787/research/benchmarks/{id}/posts?buzz=1"
```

### Step 3: パターン分析取得
```bash
curl -s "http://localhost:8787/research/benchmarks/{id}/analysis"
```

### Step 4: バズ投稿のスコア計算+要素構造化
各バズ投稿のエンゲージメントスコアを算出:
```
スコア = いいね×1 + リプライ×5 + リポスト×4 + 引用×3
```

構造化する軸:
- フォーマット（テキストのみ/画像付き/カルーセル/動画）
- 文体（問いかけ型/宣言型/ストーリー型/煽り型）— 問いかけ型がリプライ誘発
- 長さ（短文/中文/長文）
- 投稿時間帯 — ゴールデンアワー効果の検証
- テーマ・トピック
- CTA種類
- 会話誘発の仕掛け（リプライ×5に直結する要素）
- ハッシュタグパターン

### Step 5: 競合間比較（複数ベンチマークの場合）

### Step 6: 自分のアカウントとの差分
自分の投稿分析結果があれば参照:
```bash
curl -s "http://localhost:8787/chat/skill-context?skill_name=post-analyze"
```

### Step 7: スキルコンテキスト保存
```bash
curl -s -X POST http://localhost:8787/chat/skill-context \
  -H "Content-Type: application/json" \
  -d '{"skill_name":"research-competitor","output_summary":{...}}'
```
